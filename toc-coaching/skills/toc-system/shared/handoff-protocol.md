# Handoff Protocol: State Transfer Between TOC Flows

The TOC Thinking Processes have a natural sequence:

```
CRT → EC → FRT → PRT/TT
```

- CRT identifies root causes → EC resolves the core conflict maintaining them
- EC produces an injection → FRT validates it and hunts for negative branches
- FRT validates the solution → PRT/TT plans implementation
- Five Focusing Steps can run standalone or feed into any of the above (policy constraints often lead to EC; physical constraints with clear solutions lead to FRT or PRT/TT)

## What to Carry Forward

At every transition, summarize in plain language and get explicit confirmation from the user before entering the next flow.

**CRT → EC:** The root cause(s), the core conflict embedded in them, the full UDE list, and key evidence. Frame the conflict clearly: "On one hand you need X, on the other you need Y."

**EC → FRT:** The injection, which assumption it invalidates and why, the full cloud structure, and the UDE list — the FRT must verify the injection resolves these.

**FRT → PRT/TT:** All validated injections (primary + trims for negative branches), the desirable effects, remaining risks, and known implementation constraints.

**Five Steps → other flows:** The identified constraint (physical or policy), exploitation and subordination status, and why the recommended next flow is appropriate.

## Rules

1. **Always summarize before transitioning.** Present the handoff in plain language and get confirmation before entering the next flow.
2. **Preserve the UDE list.** The original UDEs are the scorecard for the entire process. Every subsequent flow should reference them.
3. **Carry forward open questions.** Don't let unresolved items silently disappear.
4. **Allow backtracking.** If a later flow reveals an earlier conclusion was wrong, return to the earlier flow with new information. This is the process working as designed.
5. **Check confidence.** Ask how confident the user is before proceeding. Low confidence means revisit, not push forward.
6. **Don't force the sequence.** Users may jump between flows, start in the middle, or use only one tool.
